# What do you want to do?

Sovereign has different starting points for people using a workspace, operators
running one, and developers extending it.

## Use Sovereign

Find an instance, understand who operates it, review its access policy, and learn
the workspace.

[Start as a user](./users)

## Host Sovereign

Evaluate the operational responsibilities, deploy an instance, and prepare it
for real users.

[Start as an operator](./operators)

## Build an app

Create a plugin and use Sovereign's shared platform capabilities for a custom
workflow.

[Start as an app developer](./developers)
