# Host your own instance

Operating Sovereign gives you control of the deployment and responsibility for
the people and data that depend on it.

## Evaluate the fit

Before inviting users, decide:

- Who the instance is for and how they gain access.
- Which apps and workflows it must support.
- Where it will run and who maintains that infrastructure.
- How email, TLS, storage, backups, monitoring, and upgrades will work.
- Which privacy, retention, acceptable-use, and support policies apply.

## Deploy an evaluation instance

The [self-hosting guide](/self-hosting) covers supported deployment paths and
configuration. Begin with test data and a non-critical environment while you
learn the operator controls.

## Prepare for production

Do not expose an evaluation deployment to real users until you have reviewed:

- [Security and secret requirements](/security)
- [Upgrade behavior](/upgrade)
- [Troubleshooting and health checks](/troubleshooting)
- Backup and restore requirements in the self-hosting documentation
- The access, support, and data policies you will communicate to users

## Publish responsibly

If you want an instance listed publicly, it needs a clear operator identity,
access policy, privacy link, support channel, app list, and verification date.
The initial directory does not accept automatic or unreviewed submissions.
