## Reference implementations

The [apps overview](/product/apps#how-these-appear-in-the-repository) maps the
platform's `plugins/` workspace. Built-in plugins demonstrate the shared shell,
while Tasks, Shopper, Plainwrite, and Wallet show first-party domain apps at
different levels of scope and maturity. The `example-*` plugins are narrower SDK
and shell references.

Use **app** in user-facing copy and **plugin** for manifests, APIs, database
schemas, packages, and developer documentation.
